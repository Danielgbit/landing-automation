# Reglas del proyecto

## Lenguaje
- Todo el código y comentarios deben estar en INGLES.
- Los nombres de variables, funciones y archivos deben estar en INGLES.

## Estilo de código
- Código simple y explícito.
- Evitar abstracciones innecesarias.
- Una función = una responsabilidad.

## Comentarios
- TODA función debe tener comentario explicando:
  - Qué hace
  - Qué recibe
  - Qué retorna
- Los comentarios son obligatorios incluso si la función es simple.

## Arquitectura
- Next.js App Router usando src/app
- Las páginas NO contienen lógica compleja.
- La lógica vive en:
  - api/
  - lib/
  - services/

## Backend
- Las API Routes deben:
  - Validar datos
  - Manejar errores
  - Retornar JSON consistente

## Supabase
- Solo se usa en demos avanzadas.
- Nunca usar Supabase en la landing simple.

## IA
- La IA se ejecuta SOLO en endpoints API.
- La IA NO decide lógica de negocio.
- La IA solo responde usando contexto controlado.

## Automatizaciones
- n8n se usa solo como orquestador.
- La lógica principal vive en Next.js.
